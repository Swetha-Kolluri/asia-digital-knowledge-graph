## ⚡ One-Time Setup (5 minutes)

### Step 1 — Create a GitHub repository

1. Go to [github.com/new](https://github.com/new)
2. Repository name: e.g. `asia-digital-knowledge-graph`
3. Set visibility to **Public** (required for free GitHub Pages)
4. Click **Create repository**

### Step 2 — Push this folder to GitHub

Open **Git Bash** (or any terminal) in the `GF Work/Test` folder:

```bash
git init
git add .
git commit -m "initial: Asia Digital Knowledge Graph"
git branch -M main
git remote add origin https://github.com/<YOUR_USERNAME>/<REPO_NAME>.git
git push -u origin main
```

### Step 3 — Enable GitHub Pages

1. In your repo, go to **Settings → Pages**
2. Under **Source**, select:
   - Branch: `main`
   - Folder: `/ (root)`
3. Click **Save**
4. GitHub shows your URL: `https://<username>.github.io/<repo-name>/`

> The first build takes ~2 minutes. Refresh until the page appears.

### Step 4 — Verify the daily automation

1. Go to **Actions** tab in your repo
2. You should see the workflow **"Daily Knowledge Graph Update"**
3. Click **Run workflow** → **Run workflow** to test it manually right now
4. Watch it complete — `index.html` gets regenerated and pushed

## 🔄 How It Works

```
Every day at 06:00 UTC
        │
        ▼
GitHub Actions runner (Ubuntu)
        │
        ▼
scripts/fetch_news.py
  ├── Queries Google News RSS for each country + keyword
  ├── Classifies articles by theme (Digital ID / DPI / AI / Data Exchange / etc.)
  ├── Falls back to curated seed articles if live fetch is sparse
  ├── Generates index.html (self-contained D3.js knowledge graph)
  └── Commits + pushes index.html to main branch
        │
        ▼
GitHub Pages serves index.html at your public URL
```

## ⚙️ Configuration

To change countries, keywords, or query frequency, edit `scripts/fetch_news.py`:

| Variable | Purpose |
|---|---|
| `COUNTRIES` | Countries and their Google News search queries |
| `THEME_KEYWORDS` | Keywords used to classify articles into themes |
| `SEED_ARTICLES` | Curated fallback articles when live fetch is sparse |
| `cron: '0 6 * * *'` | Schedule (daily-update.yml) — change timing here |
